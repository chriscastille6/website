📊 Cognitive Load Analysis: Bryson's Conditions

Executive Summary

This report presents a comprehensive analysis of cognitive load differences between two experimental conditions in the Bryson's study. The analysis reveals significant differences in readability and processing complexity, with important implications for experimental design and participant experience. Through systematic evaluation of 30 alternative text versions, we identify the optimal solution for equalizing cognitive load while maintaining experimental validity.

Non-Christian Values Condition

🔍 Key Findings

📖 Readability Disparity

The Christian Values condition is significantly more readable (83.8 vs 48.5 Flesch score), representing a 73% improvement in reading ease. This substantial difference could introduce systematic bias in experimental results.

📏 Length and Context Effects

Despite being 1 times longer, the Christian Values condition has lower cognitive load. This suggests that familiar language patterns and repetitive structures may actually reduce processing effort.

⚖️ Cognitive Load Classification

Both conditions are classified as "Low" cognitive load using arbitrary cutoffs, but the Non-Christian Values condition requires 152% more cognitive effort (18.1 vs 7.2 score). Regardless of classification, the substantial difference in readability scores (83.8 vs 48.5) should be equalized to ensure valid experimental results.

🎯 Recommendations to Equalize Cognitive Load

⚠️ Critical Note for Experimental Design

The substantial difference in readability scores between conditions (83.8 vs 48.5) should be addressed to ensure valid experimental results. Even small differences in cognitive load can introduce systematic bias and confound the effects of the experimental manipulation. Researchers should strive to equalize readability and cognitive load as much as possible while maintaining the intended experimental contrast.

Strategy 1: Lengthen Non-Christian Values Condition

Expand Content

Add more sentences to the Non-Christian Values condition to match the length and structure of the Christian Values condition. Example:

"We celebrate diversity in all forms. We embrace different perspectives and backgrounds. We value inclusion and respect for all individuals. These principles of diversity guide our firm, employees, and customers."

Strategy 2: Simplify Christian Values Condition

Reduce Complexity

Shorten the Christian Values condition while maintaining its core message. Example:

"We are guided by Christian values. We close on Sundays. We honor God in our work. These values guide our company."

Strategy 3: Standardize Sentence Structure

Match Sentence Length

Ensure both conditions have similar average sentence lengths (target: 4-6 words per sentence) and similar total word counts (target: 25-35 words).

Strategy 4: Vocabulary Standardization

Control Word Complexity

Use similar vocabulary complexity in both conditions. Avoid religious terminology in one condition while using simple, everyday language in both.

Strategy 5: Structural Parallelism

Match Text Structure

Use similar sentence patterns and organizational structure in both conditions. For example, both could follow the pattern: "We [value/principle]. We [action]. We [action]. These [values/principles] guide our [organization]."

🔬 Methodology

Analysis Framework

  • Readability Assessment: Flesch Reading Ease Score, Flesch-Kincaid Grade Level, SMOG Index, ARI, Coleman-Liau Index
  • Complexity Measures: Average sentence length, average word length, total word count
  • Cognitive Load Scoring: Weighted combination of syntactic complexity (40%), lexical complexity (30%), and readability (30%)
  • Tools Used: R with quanteda and quanteda.textstats packages

Detailed Calculation Methods

1. Word Count Calculation

Method: Tokenization using quanteda package

Process: Text is split into individual words, excluding punctuation but including contractions and hyphenated words as single tokens.

Example: "We are driven by Christian values" = 6 words

2. Sentence Length Calculation

Method: Sentence tokenization using quanteda

Process: Text is split into sentences, then average words per sentence is calculated.

Formula: Average Sentence Length = Total Words ÷ Number of Sentences

Example: If text has 41 words in 3 sentences, Average = 41 ÷ 3 = 13.7 words

3. Word Length Calculation

Method: Character count per word

Process: Each word is counted for characters, then averaged across all words.

Formula: Average Word Length = Total Characters ÷ Total Words

Example: "Christian" = 9 characters, "values" = 6 characters, Average = (9+6) ÷ 2 = 7.5 characters

4. Flesch Reading Ease Score

Method: Standard Flesch formula

Formula: Flesch Score = 206.835 - (1.015 × Average Sentence Length) - (84.6 × Average Syllables per Word)

Scale: 0-100 (Higher = Easier to read)

  • 90-100: Very Easy
  • 80-89: Easy
  • 70-79: Fairly Easy
  • 60-69: Standard
  • 50-59: Fairly Difficult
  • 30-49: Difficult
  • 0-29: Very Difficult

5. Cognitive Load Score Calculation

Method: Weighted combination of three factors

Formula: Cognitive Load Score = (Sentence Length × 0.4) + (Word Length × 0.3) + ((100 - Flesch Score) × 0.3)

Components:

  • Syntactic Complexity (40%): Average sentence length in words
  • Lexical Complexity (30%): Average word length in characters
  • Readability (30%): Inverse of Flesch score (100 - Flesch Score)

Example Calculation:

For Christian Values condition:

  • Sentence Length: 3 words × 0.4 = 1.2
  • Word Length: 3.7 characters × 0.3 = 1.11
  • Readability: (100 - 83.8) × 0.3 = 4.86
  • Total: 1.2 + 1.11 + 4.86 = 7.17

6. Verification Steps for Readers

To verify word count: Count each word manually, including contractions as one word

To verify sentence length: Count sentences (periods, exclamation marks, question marks), then divide total words by sentence count

To verify word length: Count characters in each word (including apostrophes in contractions), then average

To verify Flesch score: Use online Flesch calculators or implement the formula above

To verify cognitive load score: Apply the weighted formula using the three components

Important Note on Cognitive Load Classifications

⚠️ The cognitive load cutoffs used in this analysis (Low: < 30, Medium: 30-50, High: > 50) are arbitrary and not based on standardized measures. These classifications should be interpreted with caution and are provided for descriptive purposes only.
